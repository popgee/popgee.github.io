宝塔面板报错[url=]/www/server/nginx/sbin/nginx: error while loading shared libraries: libluajit-5.1.so.2: cannot

发表于2021-12-08|更新于2021-12-08
|浏览量:

宝塔面板报错[url=]/www/server/nginx/sbin/nginx: error while loading shared libraries: libluajit-5.1.so.2: cannot

本地系统是CentOS 7.9，宝塔面板安装Nginx 1.20，安装完成后无法添加网站，也无法启动Nginx，两个都报错[url=]/www/server/nginx/sbin/nginx: error while loading shared libraries: libluajit-5.1.so.2: cannot open shared object file: No such file or directory。提示需要先把报错解决了才能添加网站。

百度了一番，将libluajit-5.1.so.2所在的文件夹路径添加到/etc/ld.so.conf，然后使用命令/sbin/ldconfig –v更新配置，一样无法启动。

再尝试安装低版本的Nginx 1.7，成功运行，没有报错。
